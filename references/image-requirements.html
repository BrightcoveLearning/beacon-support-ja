ビデオクラウド画像

ビデオを Video Cloud に取り込むと、2 つの画像が自動的に作成され、ビデオのプロパティに表示されます。作成された2つの画像は ポスター と サムネイル。イメージは、Studio の [ イメージ ] セクションに表示されます。これらのアップロード画像は Brightcove Beacon で使用しますが、適切に機能するためには、デフォルトの取り込みプロファイルを変更する必要があります。次の警告ボックスに表示される手順は、変更を行う方法を示しています。変更を行わないと、アップロードしている縦長の画像はサムネイルのサイズが維持され、Brightcove Beacon アプリでは非常に貧弱に見えます。

デフォルトの取り込みプロファイルを変更する必要があります。以下の手順に従います。
  1. Studioで「管理者」→「プロファイルの取り込み」に移動します。
  2. デフォルトのアップロードラベルが付いた現在のデフォルトプロファイルのタイトルをクリックします。
  3. [複製]ボタンをクリックします。
  4. イメージが定義されている場所を特定します。正確な JSON は、複製したプロファイルによっては、以下に示すものとわずかに異なる場合があります。
  5. に示すように、ポスター画像とサムネイル画像の両方の幅と高さを変更します。

    古い設定

    {
  "media_type": "image",
  "format": "jpg",
  "label": "poster",
  "width": 1280,
  "height": 720,
  "id": "559a8b0ee4b07cbb40be7d22"
},
{
  "media_type": "image",
  "format": "jpg",
  "label": "thumbnail",
  "width": 160,
  "height": 90,
  "id": "559a8b0ee4b07cbb40be7d23"
}

    新しい設定

    {
  "media_type": "image",
  "format": "jpg",
  "label": "poster",
  "width": 4000,
  "height": 2250,
  "id": "559a8b0ee4b07cbb40be7d22"
},
{
    "media_type": "image",
    "format": "jpg",
    "label": "thumbnail",
    "width": 3000,
    "height": 2000,
    "id": "559a8b0ee4b07cbb40be7d23"
}
  6. [保存]をクリックします。
  7. 次に、[ これを既定のプロファイルにする ] リンクをクリックします。
  8. プロフィールがアップロードのデフォルトとしてラベル付けされていることを確認します。
これで、Video CloudStudioを使用してBrightcove Beacon の画像をアップロードする準備が整いました。

Studio インターフェイスを使用して 2 つのイメージをアップロードします。あなたのビデオを表す画像を決定し、上記の要件を満たす適切なスクリーンショットを撮ります。次に、2 つの画像を Video Cloud にアップロードします。

NS ポスター と サムネイル BrightcoveBeaconで使用される画像のプレースホルダーです。 Video Cloud Studioに表示される画像の推奨事項に従わないでください！ 代わりに、次のガイドラインを使用してください。

ポスター サムネイル
風景画像 (16:9):
最小構成 幅:1000px 高さ:563px
[最大] 幅:4000px 高さ:2250px
ポスター画像 (2:3)
最小構成 幅:1000px 高さ:1500px
[最大] 幅:2000px 高さ:3000px

スクリーンショットの左側の画像は、ポートレート画像の使用例です。非常にピクセル化された画像がある場合は、デフォルトの取り込みプロファイルを更新する必要があることがわかります。

コーポレートロゴ

Brightcove Beacon エクスペリエンスで使用するには、企業ロゴを提供する必要があります。画像については、次のガイドラインに従ってください。

仕様 値
[ 形式] png
アスペクト比 1:1
サイズ 1500px 1500px

Brightcove Beacon で働く

Brightcove Beacon に情報を入力する際には、非テキストデータの入力を求められます。これは、現在作成しているアセットの画像を意味します。これらのイメージは、次の 3 つのグループに分けることができます。

  • シリーズ/季節
  • ライブイベント
  • チャンネルロゴ

グループの画像の詳細は以下のようになります。

シリーズ/季節

シリーズ/シーズンの場合、ページは次のように表示されます（画像を入力する必要がない「サムネイル」の上に赤い「オプション」のテキストはありません）。

この画面が表示されたら、以下の要件に一致するポートレートポスター、ランドスケープポスター、およびワイドバナーのJPEG画像を提供します。サムネイルに画像を指定する必要はありません。指定しない場合、サムネイル画像はポートレートポスターから生成されます。

イメージ 要件
ポートレートポスター ポスター画像 (2:3)
最小構成 幅:1000px 高さ:1500px
[最大] 幅:2000px 高さ:3000px
風景ポスター 風景画像 (16:9):
最小構成 幅:1000px 高さ:563px
[最大] 幅:4000px 高さ:2250px
ワイドバナー 風景画像 (21:9):
最小構成 幅:1000px 高さ:428px
[最大] 幅:4000px 高さ:1714px

ライブイベント

ライブイベントの場合、次のようなページが表示されます（画像を提供する必要がない「サムネイル」と「ワイドバナー」の上に赤い「オプション」のテキストが表示されません）。

この画面が表示されたら、以下の要件に一致するポートレートポスターとランドスケープポスターのJPEG画像を提供します。サムネイルおよびワイドバナーDOの画像を提供する必要はありません。指定しない場合、サムネイル画像はポートレートポスター、ワイドバナーはランドスケープポスターから生成されます。

イメージ 要件
ポートレートポスター ポスター画像 (2:3)
最小構成 幅:1000px 高さ:1500px
[最大] 幅:2000px 高さ:3000px
風景ポスター 風景画像 (16:9):
最小構成 幅:1000px 高さ:563px
[最大] 幅:4000px 高さ:2250px

チャンネルロゴ

チャンネルロゴ画像は、サムネールの背景とイメージ（透明の有無にかかわらず）で構成され、どちらもアスペクト比 16:9 です。次の図は、チャネルロゴの作成方法を示しています。

これは、新しいチャンネルを作成するときに表示される非テキストデータフォームです。

サイズ要件は表に記載されています。

イメージ 要件
チャンネルロゴ画像 風景画像 (16:9):
必要なサイズ 幅:1000px 高さ:563px
