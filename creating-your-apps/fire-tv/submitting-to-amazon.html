概要

このドキュメントは、Brightcove Beacon アプリを Amazon アプリストアに送信する際に役立ちます。

プロセスの概要は次のとおりです。

  1. Amazonデベロッパーコンソールで新しいAndroidアプリを追加します。
  2. 新しいFire TVアプリの可用性と価格を設定します。
  3. Amazonアプリストアで、Fire TVアプリの説明文を追加します。
  4. Fire TVアプリのアイコンと画像を追加します。
  5. Fire TVアプリのコンテンツレーティングを選択します。
  6. Brightcoveが提供する APK ファイルをアップロードします。

このドキュメントの残りの部分では、上記の各手順について詳しく説明します。

アマゾンのリソース

  • アプリストアの詳細: https://developer.amazon.com/home.html (Amazonデベロッパーアカウントが必要-手数料なし)
  • Fire TV アプリ提出ガイド: https://developer.amazon.com/docs/fire-tv/submitting-your-app-to-the-amazon-appstore.html そしてより一般的には: https://developer.amazon.com/apps-and-games/app-submission
  • 承認のための推奨リードタイム:三週間

Amazonに必要な情報

  • Fire TVアプリのアイコン (1280 x 720px PNG (透明度なし))。
  • 3枚から10枚のスクリーンショットの間（1920 x 1080ピクセルJPGまたはPNG（風景、透明度なし）。
    • これらの画像は、UI で表示したときに、歪んだり、引き伸ばしたり、ピクセル化したりすることはできません。
    • アプリを送信するときにFire TVのスクリーンショットを提供するか、すべてのデバイスでアプリを表すスクリーンショットを使用してください。
    • 申請プロセスでAmazon Fire TV固有のスクリーンショットを提供しない場合、指定したスマートフォンやタブレットのスクリーンショットは、Fire TV App Storeでアプリを表すために使用されます。
  • 背景画像 1920 x 1080ピクセル JPG または PNG (横、透明度なし)。
  • Brightcoveから提供された APK パッケージ。
  • 表示タイトル。
  • 短い説明。
  • 長い説明。
  • 製品機能の箇条書き（Fire TVアプリの主要機能を10個まで）。
  • キーワード (アプリの発見性を高めるために使用される検索用語)。

Fire TV アプリをAmazonアプリストアに送信する

以下の手順に従って、Fire TVアプリをAmazonアプリストアに送信してください。

  1. Amazonデベロッパーコンソールに入ります。コンソールを使用するには、Amazonデベロッパーアカウントを作成する必要があります。
  2. Amazonアプリストアのセクションで、新しいアプリを追加ボタンをクリックします。
  3. ドロップダウンメニューから、[ Android ] を選択します。（現時点では、Brightcoveは、Fire TV アプリの提出用の Android バージョンのみをサポートしています）。
  4. 「新規アプリの提出」フォームが表示されます。
  5. アプリのタイトルを入力します。
  6. アプリ SKU を選択します。これは、自分で定義するアプリに一意の文字列です。SKU（Stock Keeping Unit）は、次のように、独自の追跡目的でアプリのIDになります。 com.amazon.example.myapp.v1。
  7. アプリのカテゴリを選択します。アプリを適切に分類することで、ユーザーはアプリをより簡単に検索できるようになり、アプリのダウンロード数が増加します。詳細については、 Amazonカテゴリーのドキュメントをご覧ください。
  8. [ 既定のサポート情報を使用する ] チェックボックスをオンにするか、Fire TVアプリのカスタマーサポート連絡先に関するさまざまな情報 (電子メール、電話、Webサイト) を紹介することができます。
  9. [ 保存 ] ボタンをクリックします。
  10. 保存すると、Fire TVアプリの設定と [ 一般情報 ] タブが表示されます。
  11. このタブでは、 Amazonによって自動的に生成されたアプリ提出APIキーと、Fire TVアプリの作成時に提供されるすべての情報を見ることができます。

在庫状況と価格情報を記入する

このセクションでは、Fire TVアプリの価格と利用可能性を設定できます。選択する価格は、Brightcove Beacon プラットフォームで設定された価格と一致する必要があります。

  1. [ 在庫状況と価格 ] タブをクリックします。
  2. 最初のセクションでは、Fire TVアプリを利用できる場所を選択できます。
    • Amazonがアプリを販売しているすべての国と地域。
    • 選択した国と地域でのみ使用できます。
  3. [ 選択した国と地域のみ ] を選択した場合、このオプションでは、Amazon ストアでFire TVアプリを検索できる国または特定の地域を選択できます。以下に示す例では、「北米」のすべてと「ヨーロッパ」の3か国がFire TVアプリに選択されています。
  4. 地域から特定の国を選択するには、 + 国/地域を選択ボタンをクリックします。
  5. 選択した地域から、Fire TVアプリを公開する国を選択します。
  6. [追加]ボタンをクリックします。
  7. 次のオプションでは、Fire TVアプリが無料か有料かを選択できます。通常、デフォルトの Brightcove Beacon アプリは [ 無料 ] に設定されています。
  8. [ 支払済 ] を選択した場合は、基準価格と通貨の種類を設定する必要があります。（Fire TV アプリに価格を割り当てる場合は、Brightcove Beacon のアカウント担当者にお問い合わせください）。
  9. （オプション）Fire TVアプリが承認されるとすぐに公開プロセスを開始するには、このフィールドを空白のままにします。公開プロセスの開始を将来の日時に延期するには、Fire TVアプリの公開日を割り当てます。
  10. [ 保存 ] ボタンをクリックします。

アプリ内購入 (IAP) サブスクリプションの作成

このセクションでは、Fire TVアプリのアプリ内購入サブスクリプションを作成する方法について説明します。

  1. 在庫状況と価格設定を保存すると、[ IAPを追加 ] (アプリ内課金) の追加フィールドが表示されます。
  2. [ IAPを追加 ] ボタンをクリックして、アプリ内購入をサブスクリプションとして追加します。
  3. [ アプリ内アイテム ] タブが表示されます。
  4. 現時点では、そこにアプリ内アイテムはありません。1 つ作成するには、[ 単一 IAP を追加 ] ボタンをクリックします。
  5. アプリ内購入には 3 種類あります。
    • 消耗品の追加:コンシューマブルには、アプリ内で販売するコンテンツのうち、アプリ内で消費されるあらゆる種類のコンテンツが含まれます。このタイプのコンテンツは、購入したデバイスでのみ使用できます。
    • エンタイトルメントを追加する:エンタイトルメントには、アクセス権を必要とするアプリ内で販売するあらゆる種類のコンテンツが含まれます。この種類のコンテンツは、お客様がAmazon Appsクライアントにログインしている場所であればどこでも利用できます。コンテンツに対するエンタイトルメントは期限切れになりません。
    • サブスクリプションの追加:サブスクリプションには、次のようなコンテンツの種類が含まれます。
      • エンタイトルメントを運ぶ
      • 一定期間にバインドされているか
      • 期間終了時に自動更新
      • カスタマーの Amazon アカウントに登録されているすべての対象端末で利用可能
      • アプリには複数のサブスクリプションを定義でき、各サブスクリプションには複数の期間を設定できます。
  6. ご希望の IAP を選択するか、Brightcove Beacon のアカウント担当者にお問い合わせいただき、お客様のコンテンツに最適なオプションについてご確認ください。
  7. この例では、[ サブスクリプション ] が選択されています。
  8. 新しいサブスクリプションにタイトルを付けます。
  9. SKUを選択します。SKU（Stock Keeping Unit）は、たとえば、独自の追跡目的でサブスクリプションのIDになります。 Subscription.gold。
  10. [ 保存 ] ボタンをクリックします。
  11. IAPを保存すると、[ サブスクリプション期間 ] タブが表示されます。
  12. サブスクリプション期間を選択します。サブスクリプションがユーザーに請求される時間を選択できます。たとえば、毎月、毎週、毎年など
  13. SKUを選択します。SKU（Stock Keeping Unit）は、たとえば、独自の追跡目的でサブスクリプション期間のIDになります。 毎月のサブスクリプション。
  14. （オプション）サブスクリプションの無料トライアルを行う場合に選択します。例は次のとおりです。7日、1ヶ月など
  15. このサブスクリプションの課金を行うかどうかを選択します。充電の場合は、[ はい、私の基本希望小売価格...] を選択します。オプションを使用します。
  16. サブスクリプションの価格と通貨の種類を入力します。
  17. [ 計算価格 ] 設定が表示され、出品者が希望小売価格を計算するか、手動で価格を設定するかを選択できます。
  18. [ 保存 ] ボタンをクリックします。
  19. [ 説明 ] タブに移動し、新しいサブスクリプションに説明を追加します。
  20. [ 保存 ] ボタンをクリックします。
  21. すべてのIAP設定が完了すると、[ アプリ内アイテムを提出 ] ボタンが有効になります。ボタンをクリックして、承認のためにIAPをAmazonに送信します。
  22. アプリ内アイテムを送信ボタンをクリックすると、 IAPアイテムがアプリ内アイテムタブのリストに表示されます。

説明情報を入力します。

このセクションでは、Amazonストアのユーザーに表示されるすべてのFire TVアプリのテキスト説明を追加できます。

  1. [ 現在のバージョン ] タブをクリックし、Fire TVアプリのメニューから [ 説明 ] タブを選択します。
  2. フォームで以下を入力します。
    • 短い説明 :モバイルデバイスに表示されるアプリの簡単な説明。
    • 長い説明 :App Store ウェブサイトの、アプリのより長い説明。
    • 製品機能の箇条書き :アプリには最大 10 個の主要機能、1 行に 1 つのフィーチャがあります。これらの機能は App Store の Web サイトに表示され、箇条書きリストとして書式設定されます。
    • （任意）キーワード :アプリの発見性を高めるために使用される検索語句、コンマまたは空白を使用して用語を区切ります。
  3. デフォルトでは、説明は英語で表示されますが、[ ローカライズされた説明を追加] ボタンをクリックすると翻訳を追加できます。
  4. フォームが表示され、追加したい翻訳言語を選択できます。
  5. [追加]ボタンをクリックします。
  6. フォームの上部で、言語フォームから選択して、各言語の説明を追加することができます。
  7. [ 保存 ] ボタンをクリックします。

完全なコンテンツレーティング情報

コンテンツレーティングセクションでは、Fire TVアプリで提示するコンテンツの種類を分類するように要求されます。Amazon の分類では、コンテンツを次の 3 つのタイプに分類します。

  • 中程度:一度またはまれに発生し、アプリの全体的な目的および/または意図の基本ではありません。
  • 強い:定期的に発生し、アプリの全体的な目的および/または意図の基本です。
  • なし:アプリでは発生しません。

被験者の種類は、暴力、学術、漫画の暴力です。

  1. [ コンテンツ評価 ] タブをクリックします。
  2. 各サブジェクトのカテゴリタイプを選択します。この情報はすべて、Fire TVアプリのコンテンツタイプに対応しています。答えがわからない場合は、Brightcove Beacon のアカウント担当者にお問い合わせください。
  3. [ 追加情報 ] セクションで、Fire TVアプリに含まれる機能に応じて [はい] または [いいえ ] を選択します。この情報はすべて、Fire TVアプリが保持する可能性のある追加情報と機能に対応しています。答えがわからない場合は、Brightcove Beacon のアカウント担当者にお問い合わせください。
  4. Fire TVアプリが個人情報を収集する場合は、プライバシーポリシーURLを追加して、ユーザーのプライバシーポリシーを確認する必要があります。
  5. [ 保存 ] ボタンをクリックします。

完全なAndroidパッケージキット（APK）ファイル情報

APKファイルセクションは、Androidアプリファイルをアップロードする場所です。APK はBrightcoveから提供されます。

  1. [ APK ファイル ] タブをクリックします。
  2. Amazon DRMを適用しますか？オプションで、[ いいえ ] を選択します。
  3. [ APK ファイル ] セクションで、右側のボックスをクリックし、Brightcoveから提供された APK ファイルをアップロードします。
  4. ファイルがアップロードされると、画面上でファイルを見ることができます。[ 表示 ] ボタンをクリックします。
  5. デバイスサポートウィンドウが開き、Fire TVアプリがサポートされるデバイスを選択できます。
  6. Amazon Fire TVタブをクリックし、すべてのデバイスを選択します。BrightcoveBeaconはAmazonFire TVでのみ使用できるため、[AmazonFireタブレットと電話]タブと[非Amazonデバイス]タブは選択しないでください。
  7. [ デバイスサポート ] ウィンドウを閉じます。
  8. [ その他の詳細の追加 ] セクションで、リストから言語サポートを少なくとも 1 つ選択します。これは、ユーザーがサポートのために通信する言語です。
  9. [ Testing Restructions ] フィールドで、ビデオストリームをテストするために必要なログイン認証情報や、異なるサブスクリプションレベルの使用方法を説明するメモなど、Amazon のテストチームに特別な指示を提供します。
  10. 「ExportCompliance」および「UseAmazonMapsRedirection」チェックボックスをオンにします。
  11. [ 保存 ] ボタンをクリックします。
  12. 保存すると、[ レビューステータス ] タブがメニューに表示されます。
  13. 最後のアクティビティや送信履歴として、Fire TVアプリに関する情報を確認できます。エラーがない場合は、画面の左上隅にある [ アプリを送信 ] ボタンをクリックして、Fire TVアプリを送信します。

これで、AmazonがFire TVアプリを承認し、Amazonアプリストアで公開するまで待つだけです。
