チャネルアクションの使用

チャネルのステータスに応じて、チャネル設定ページでさまざまなチャネルアクションボタンを使用できます。チャネル名をクリックしてチャネルアクションボタンにアクセスし、チャネルの詳細を表示します。次のチャネルアクションボタンを使用できます。

  • -チャンネルをアクティブにすると、ストリーミングで利用できるようになります。動画やライブイベントをチャンネルに追加して、クリックする必要があります [アクティブ化]ボタンが表示される前。
  • - クリック今すぐプレイチャネルのストリーミングを開始します。このオプションは、開始時刻が近づいているチャンネルでのみ使用できます。チャネルは、スケジュールされた開始時間に自動的に開始されます。
  • - クリック今すぐ停止実行中のチャネルを停止します。これにより、チャンネルが停止し、ライブストリームが終了します。チャネルがループしていて、終了日時がある場合、チャネルはその時間に自動的に停止します。このオプションは、それを早期に停止する方法を提供します。

チャネル停止時のリソースの割り当て解除

チャネルを停止すると、次のことが発生します。

  1. クラウドプレイアウトチャネルが停止すると、下書きユーザーがチャネル設定を再度変更できるようにする状態。
  2. これにより、連鎖したBCライブイベントがあればそれもクリアされます。
  3. チャネルはする必要があります有効化リソースが割り当てられ、チャネルがスケジュール州。
  4. クラウドプレイアウトチャンネルがライブストリームに対して有効になっている場合は、 RTMP取り込みの静的エントリポイントこれは、チャネルが停止および再開された回数に関係なく、チャネルの存続期間を通じて静的なままになります。

    このエントリポイントは、チャネルが起動して実行されると、入力の受け入れを開始できるようになります。

  5. 内部の問題が原因でCloudPlayoutが静的RTMP取り込みポイントを提供できない場合は、IPアドレスベースの取り込みポイントを表示しようとします。これが発生した場合は、チャネルのRTMP取り込みポイントの変更を定期的に確認することをお勧めします。
  6. 注:チャネルをアクティブ化した後、Cloud Playoutが静的取り込みポイントの新しいIPアドレスを公開するのに平均1〜2分かかります。そのため、チャンネルがアクティブ化された直後に1〜2分以内に開始され、ライブイベントがチャンネルの再生リストの最初のアセットである場合、取り込みポイントが解決できない可能性があります。静的取り込みポイントのIPアドレスが公開されるとすぐに解決可能になります。
  7. 注:チャネルが最初の実行でライブ入力で実行されて停止した後、チャネルのライブ入力をオフにした場合、既存のプログラムにはまだ失敗するライブイベントがある可能性があります。チャンネルを停止した後にライブストリームを無効にする場合は、チャンネルコンテンツからライブイベントを削除してから再開する必要があります。
